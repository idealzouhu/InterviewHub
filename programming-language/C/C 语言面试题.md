### 关键字 static 的作用是什么？

static是 C 语言中一个非常重要但容易混淆的关键字，用于限制生命周期和可见性。它有三种不同的作用，取决于使用的上下文。

(1) **在函数内部：静态局部变量**

**特点**：

- 只初始化一次（程序开始时）
- 生命周期是整个程序运行期间
- 作用域仍限于该函数内部
- 保持值在函数调用之间

```c
void func() {
    int normal_var = 0;       // 普通局部变量
    static int static_var = 0; // 静态局部变量
    
    normal_var++;
    static_var++;
    
    printf("normal: %d, static: %d\n", normal_var, static_var);
}

int main() {
    func();  // 输出: normal: 1, static: 1
    func();  // 输出: normal: 1, static: 2
    func();  // 输出: normal: 1, static: 3
    return 0;
}
```



**(2) 在文件作用域（全局变量前）：内部链接**

**特点**：

- 限制变量/函数只在**当前文件**内可见
- 避免命名冲突
- 实现信息隐藏（类似私有成员）

```c
// file1.c
static int file1_var = 100;  // 只在 file1.c 中可见

void file1_func() {
    printf("%d\n", file1_var);  // 可以访问
}

// file2.c
extern int file1_var;  // 错误！链接器找不到 file1_var
```



**(3) 在函数声明前：内部链接**

```c
// utils.c
static void helper_function() {  // 只在当前文件可用
    printf("内部辅助函数\n");
}

void public_function() {  // 可以被其他文件调用
    helper_function();  // 可以调用
}

// main.c
void helper_function();  // 错误！链接器找不到
void public_function();  // 正确
```







### extern 的作用是什么？

extern 表示的变量或者函数声明，其定义在另一个文件中。其主要是告诉告诉编译器定义在另一个文件中。

其作用主要有：

- **避免多重定义**：确保内存只分配一次
- **提高可维护性**：声明和定义分离
- **链接器友好**：明确指定变量在哪里定义



### 关键字 const 有什么用？

`const` 是 C 语言中非常重要的关键字，它的核心作用是声明只读性，告诉编译器"这个对象不应该被修改"。

**(1) 基本用法:  定义常量**

```c
const int MAX_SIZE = 100;      // 整型常量
const double PI = 3.1415926;   // 浮点常量
const char* NAME = "Program";  // 字符串常量
```

**(2) 指针与 const 的组合**

- 情况1：指向常量的指针

  ```c
  const int* p;     // p 可以指向不同的地址，但不能通过 p 修改所指向的值
  int x = 10;
  p = &x;
  *p = 20;         // 错误！不能通过 p 修改
  x = 20;          // 正确，x 本身不是 const
  ```

- 情况2：常量指针

  ```c
  int* const p;     // p 的指向不能改变，但可以通过 p 修改值
  int x = 10, y = 20;
  p = &x;          // 正确，初始化时可以赋值
  p = &y;          // 错误！p 的指向不能改变
  *p = 30;         // 正确，可以修改指向的值
  ```

- 情况3：指向常量的常量指针

  ```c
  const int* const p;  // p 的指向不能变，也不能通过 p 修改值
  int x = 10;
  p = &x;             // 错误！指向不能变（如果已初始化则正确）
  *p = 20;            // 错误！值不能改
  ```



### 深入探讨变量的声明与定义的区别？

在编程中，变量的**声明**指的是告知编译器变量的名称和类型，但不分配内存空间。声明可以多次，常见于头文件中，用于模块间的接口声明。使用`extern`关键字声明的变量，意味着其定义在别处，通常在另一个文件中。

相对地，**定义**则是创建一个具有存储空间的变量实例。定义只能有一次，通常在源文件中，确保为变量分配内存空间。例如，全局变量和局部变量的定义就是分配内存并初始化的过程。



### const与 #define 相比，有何优点？

const与 #define相比有多个重要优点，主要体现在类型安全、调试友好、作用域控制等方面。

- const 常量有数据类型，可以执行类型安全检查。而 #define 宏常量没有数据类型，不能执行安全检查。

- #define 在调试时只能看到预处理后的值。const 在调试时可见。

  ```c
  #define DEBUG_MODE 1
  #define MAX_RETRY 3
  #define TIMEOUT 5000
  
  if (DEBUG_MODE) {  // 预处理后：if (1) {
      printf("Debug\n");
  }
  
  // 调试时只能看到：
  // if (1) {
  //     printf("Debug\n");
  // }
  // 看不到原始的 DEBUG_MODE
  ```

- #define 是全局的，const 有正常的作用域



### sizeof和strlen的区别？

| 特性           | `sizeof`                        | `strlen`                    |
| -------------- | ------------------------------- | --------------------------- |
| **性质**       | 编译时**运算符**                | 运行时**库函数**            |
| **作用**       | 获取对象/类型的内存大小（字节） | 获取字符串长度（字符数）    |
| **参数**       | 类型或表达式                    | 字符串指针（`const char*`） |
| **包含结束符** | ✅ 包含 `\0`                     | ❌ 不包含 `\0`               |
| **时间复杂度** | O(1)（编译时计算）              | O(n)（遍历字符串直到 `\0`） |
| **头文件**     | 不需要（语言内置）              | 需要 `#include <string.h>`  |

例如

```
char str1[] = "Hello";      // 字符数组
char* str2 = "Hello";       // 字符串指针
char str3[100] = "Hello";   // 大数组

printf("sizeof(str1) = %zu\n", sizeof(str1));  // 6 (数组大小)
printf("sizeof(str2) = %zu\n", sizeof(str2));  // 4或8 (指针大小)
printf("sizeof(str3) = %zu\n", sizeof(str3));  // 100 (数组大小)

printf("strlen(str1) = %zu\n", strlen(str1));  // 5
printf("strlen(str2) = %zu\n", strlen(str2));  // 5
printf("strlen(str3) = %zu\n", strlen(str3));  // 5
```





### 区分 strcpy、sprintf与memcpy？

- `strcpy`：专门用于**复制以 `'\0'`结尾的字符串**。它从源地址开始复制，直到遇到源字符串的结束符 `'\0'`为止，并将这个 `'\0'`也复制到目标地址。
- `sprintf`： 将格式化的**多种类型的数据**（整数、浮点数、字符串等）按照指定的格式“打印”到一个字符串缓冲区中。无论格式化和参数处理结果如何，`sprintf`都会在写入所有内容后，自动在末尾添加一个\0。
- `memcpy`： 用于复制任意**原始内存块**。它不关心内存里是什么内容（字符串、结构体、数组、二进制数据等），只按照指定的字节数进行精确复制。

| 特性               | `strcpy`          | `sprintf`                  | `memcpy`                                    |
| ------------------ | ----------------- | -------------------------- | ------------------------------------------- |
| **主要目的**       | 复制字符串        | 格式化字符串               | 复制内存块                                  |
| **操作对象**       | 字符串（`char*`） | 缓冲区 + 各种类型数据      | 任意内存（`void*`）                         |
| **需要指定长度？** | 否（自动找`\0`）  | 否（但格式指定了数据转换） | **是**（核心参数）                          |
| **添加 `\0`吗？**  | **否**（自动）    | **是**（自动）             | **否**（只复制指定字节）                    |
| **关键风险**       | 目标缓冲区太小    | 格式化结果太长             | 内存区域重叠                                |
| **安全版本**       | `strncpy`         | **`snprintf`**             | （无内置“安全”版本，但可用`memmove`防重叠） |
| **性能考量**       | 需逐字节检查`\0`  | 需解析格式，较慢           | 通常最快，直接按字节复制                    |





### 怎么处理内存级别的数据？

**(1) 数据类型**

操作字节首选 `uint8_t`，或者 `unsigned char`

```
#include <stdint.h>

int8_t    // 有符号 1 字节
uint8_t   // 无符号 1 字节 ✅（最常用）
int16_t
uint16_t
int32_t
uint32_t
int64_t
uint64_t



char c;          // 是否带符号取决于编译器
unsigned char uc;


sizeof() // 返回 size_t，用于数组长度、内存大小
```

**(2) 按位操作符**

| 操作符 | 含义     | 示例     |
| ------ | -------- | -------- |
| `&`    | 按位与   | `a & b`  |
| `|`    | 按位或   | `a | b`  |
| `^`    | 按位异或 | `a ^ b`  |
| `~`    | 按位取反 | `~a`     |
| `<<`   | 左移     | `a << 3` |
| `>>`   | 右移     | `a >> 2` |

常见用途：

```
// 取第 n 位
(x >> n) & 1

// 设置第 n 位
x |= (1 << n);

// 清除第 n 位
x &= ~(1 << n);
```

**(3) 内存操作函数**

| **函数**    | **作用**   |
| :---------- | :--------- |
| `memcpy()`  | 拷贝字节   |
| `memmove()` | 可重叠拷贝 |
| `memset()`  | 填充字节   |
| `memcmp()`  | 比较字节   |





### 详述C/C++程序编译时的内存分配

C/C++程序内存分配包括：

1. **静态存储区**：存储全局变量、静态变量、常量。
2. **栈区**：存储函数局部变量、函数参数。
3. **堆区**：通过`malloc`/`new`分配，由程序员管理。
